# Lyhyt kirjoitus
<p>GitHub Actions on GitHubin tarjoama CI/CD (Continuous Integration/Continuous Deployment) -ratkaisu, joka mahdollistaa automaattisten työnkulkujen luomisen sovelluskehityksen eri vaiheisiin. Sen avulla voidaan esimerkiksi ajaa automaattisia testejä, varmistaa koodin laatu ja julkaista sovellus automaattisesti eri ympäristöihin. GitHub Actions on erityisen kätevä GitHubin sisäänrakennettuna ominaisuutena. Vastaavia CI/CD-ratkaisuja ovat myös muun muassa Jenkinsja Docker, jotka kaikki mahdollistavat joustavat työnkulut.
</p>
<P>
CI/CD-työnkulku sisältää usein erilaisia vaiheita, kuten koodin laadun tarkastuksia, testejä ja automaattisia julkaisuja. Työnkulut käynnistyvät automaattisesti, kun GitHub tunnistaa muutoksia määritellyissä haaroissa. Jekyll-sivuston yhteydessä voit luoda Actions-työnkulun, joka asentaa tarvittavat riippuvuudet, rakentaa Jekyll-sivuston ja julkaisee sen suoraan GitHub Pagesiin.
</P>
